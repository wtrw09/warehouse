# 仓库管理系统使用说明

## 系统概述

仓库管理系统是一个基于Vue.js和Element Plus构建的现代化仓库管理解决方案，提供完整的物料管理、出入库管理、基础数据维护和系统管理功能。

## 1. 登录系统

### 1.1 登录界面
- 访问系统后首先进入登录页面
- 输入用户名和密码进行身份验证
- 支持快捷键操作：Ctrl+Shift+A 可快速跳转到注册页面

### 1.2 登录流程
1. 在用户名输入框中输入您的用户名
2. 在密码输入框中输入密码
3. 点击"登录"按钮或按Enter键提交
4. 系统验证通过后自动跳转到主页面

### 1.3 注意事项
- 用户名和密码区分大小写
- 登录失败会保留已输入的用户名，方便重新尝试
- 开发环境下会自动保存用户名字段到本地存储
- 登录成功后自动启动Redis状态监控

## 2. 基础数据管理

### 2.1 器材管理
**功能描述：** 管理仓库中的各类器材信息

**操作步骤：**
1. 点击"新增器材"按钮添加新器材
2. 填写器材名称、型号、规格、单位等基本信息
3. 设置器材分类和库存预警值
4. 支持批量删除选中的器材记录

**数据输入要点：**
- 器材名称必须唯一且具有描述性
- 规格型号要准确反映器材特性
- 单位选择要符合实际使用习惯
- 库存预警值设置要合理

### 2.2 供应商管理
**功能描述：** 维护供应商信息，支持数据导入

**操作步骤：**
1. 点击"新增供应商"添加供应商信息
2. 填写供应商名称、联系人、电话、地址等
3. 支持Excel模板导入批量数据
4. 可下载导入模板进行数据准备

**数据输入要点：**
- 供应商名称要完整且唯一
- 联系方式要准确有效
- 导入前检查数据格式是否符合模板要求
- 批量导入时注意数据去重

### 2.3 客户管理
**功能描述：** 管理客户信息，支持数据导入导出

**操作步骤：**
1. 点击"新增客户"添加客户信息
2. 填写客户名称、联系人、联系方式等
3. 支持Excel导入和模板下载
4. 可对客户信息进行批量操作

**数据输入要点：**
- 客户名称要规范统一
- 联系信息要完整准确
- 客户分类要合理设置

### 2.4 其他基础数据
- **库位管理：** 管理仓库中的存储位置
- **专业管理：** 管理器材所属的专业分类
- **设备信息：** 管理仓库设备信息

## 3. 物料出入库管理

### 3.1 入库管理
**功能描述：** 管理物料入库流程，包括入库单创建、审核和执行

**操作流程：**
1. 进入入库管理界面
2. 点击"新增入库单"创建新入库单
3. 选择供应商、入库类型
4. 添加入库物料明细
5. 提交审核或直接执行入库

**数据输入要点：**
- 入库单号要符合编号规则
- 供应商选择要准确
- 物料明细要完整，包括数量、批次等信息
- 入库时间要准确记录

### 3.2 出库管理
**功能描述：** 管理物料出库流程，包括出库单创建和执行

**操作流程：**
1. 进入出库管理界面
2. 点击"新增出库单"创建新出库单
3. 选择客户、出库类型
4. 添加出库物料明细
5. 提交审核或直接执行出库

**数据输入要点：**
- 出库单号要唯一且规范
- 客户选择要准确
- 物料出库数量不能超过当前库存
- 出库原因要明确记录

### 3.3 库存查询
**功能描述：** 实时查询库存状态，支持多种筛选条件

**功能特点：**
- 可按专业、器材类型等条件筛选
- 支持隐藏无库存器材
- 可导出Excel报表
- 实时显示库存数量和位置

**使用技巧：**
- 使用专业筛选快速定位特定类别器材
- 隐藏无库存功能可简化界面显示
- 导出Excel便于数据分析和备份

**配置项目：**

- 业务规则配置

- 数据备份策略

**注意事项：**
- 修改配置前确认权限
- 重要配置修改要谨慎
- 修改后及时保存

### 4.2 物料编码级别配置
**功能描述：** 配置物料编码的层级结构

**功能特点：**
- 定义编码规则和层级
- 设置各级编码的长度和规则
- 确保编码的唯一性和规范性

### 4.3 个人设置
**功能描述：** 用户个性化设置

**设置项目：**
- 界面主题设置
- 个人偏好配置
- 密码修改
- 通知设置

## 5. 账户管理

### 5.1 用户管理
**功能描述：** 管理系统用户账户

**操作权限：** 需要AUTH-read和AUTH-edit权限

**功能特点：**
- 新增、编辑、删除用户
- 批量分配角色
- 用户状态管理（启用/禁用）
- 用户权限设置

**数据输入要点：**
- 用户名要唯一且易于识别
- 密码设置要符合安全要求
- 角色分配要基于最小权限原则
- 用户信息要完整准确

### 5.2 角色管理
**功能描述：** 管理系统角色和权限分配

**功能特点：**
- 创建和管理角色
- 配置角色权限
- 角色继承和层级关系
- 权限细粒度控制

**权限配置要点：**
- 权限设置要基于业务需求
- 避免权限过度分配
- 定期审查权限配置
- 重要权限要严格控制

### 5.3 权限管理
**功能描述：** 细粒度的权限控制管理

**权限分类：**
- BASE-read：基础数据读取权限
- BASE-edit：基础数据编辑权限
- AUTH-read：系统管理读取权限
- AUTH-edit：系统管理编辑权限

### 5.4 登录记录管理
**功能描述：** 查看用户登录历史记录

**功能特点：**
- 记录登录时间、IP地址
- 登录成功/失败状态
- 支持按时间范围查询
- 异常登录监控

## 6. 数据输入方法和注意事项

### 6.1 数据输入基本原则

**准确性原则：**
- 所有输入数据必须真实准确
- 数量、金额等数值型数据要仔细核对
- 日期时间要准确记录

**完整性原则：**
- 必填字段必须完整填写
- 关联数据要建立正确关系
- 历史记录要完整保存

**规范性原则：**
- 编码要符合系统规范
- 名称要统一规范
- 数据格式要标准统一

### 6.2 批量数据导入

**导入前准备：**
1. 下载对应的Excel模板
2. 按照模板格式准备数据
3. 检查数据的完整性和准确性
4. 去除重复数据和无效数据

**导入注意事项：**
- 文件格式必须为.xlsx或.xls
- 数据量较大时分批导入
- 导入前备份现有数据
- 导入后验证数据完整性

### 6.3 数据验证规则

**基础数据验证：**
- 名称不能为空且要唯一
- 编码要符合规则且唯一
- 数值型数据要在合理范围内
- 日期数据要有效

**业务数据验证：**
- 库存数量不能为负数
- 出入库数量要合理
- 关联数据要存在
- 业务逻辑要符合规则

## 8. 常见问题处理

### 8.1 登录问题
- **忘记密码：** 联系系统管理员重置
- **账户锁定：** 联系管理员解锁
- **权限不足：** 申请相应权限

### 8.2 数据操作问题
- **数据无法保存：** 检查必填字段和权限
- **导入失败：** 检查数据格式和模板
- **查询无结果：** 检查筛选条件

### 8.3 系统性能问题
- **响应缓慢：** 减少同时操作的数据量
- **页面卡顿：** 清理浏览器缓存
- **功能异常：** 刷新页面或重新登录
